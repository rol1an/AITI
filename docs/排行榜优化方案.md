# 排行榜统计优化方案

## 背景

原实现每次访问 `/api/stats/*` 都对 `submissions` 表做 `GROUP BY` 聚合查询。Cloudflare D1 单线程特性导致吞吐量受限：查询耗时 100ms 则只能达到 ~10 qps。高流量环境下不适合每次请求都实时扫表。

## 方案：预计算 + 快照表

### 架构变更

```
原架构：
  Pages Functions(/api/stats/*) → D1 submissions (GROUP BY, 每次实时聚合)

新架构：
  Pages Functions(/api/stats/*) → D1 stats_snapshot (读预计算快照)
                                 ↑
  Cron Worker (每 15min) ─────────┘
  (计算排行榜，写快照表)
```

### 核心实现

1. **快照表** (`stats_snapshot`)
   - 存储三类快照：`overview`、`archetypes`、`characters`
   - 每条记录为 key-value：key 是快照名，value_json 是预计算的 JSON

2. **Cron Worker** (`cron-worker/`)
   - 独立 Worker 项目，配置 Cron Trigger `*/15 * * * *`
   - 每 15 分钟读一次原始数据，进行 GROUP BY 聚合
   - 写结果回 `stats_snapshot`

3. **Stats API** (`functions/api/stats/*`)
   - 改为只读快照表，查询变成简单的 `SELECT * FROM stats_snapshot WHERE key = ?`
   - 响应头保留 `Cache-Control` 缓存策略

### 性能收益

| 场景 | 原方案 | 新方案 |
|------|--------|--------|
| 每次查询耗时 | ~100ms (GROUP BY) | ~1-5ms (快照表查询) |
| 理论 QPS | ~10 | ~200-1000 |
| D1 负荷 | 高频聚合 | 仅 Cron 每 15min 一次 |

## 部署步骤

### 前提条件

- Cloudflare 账户有 D1 数据库权限
- 本地已安装 Wrangler CLI

### 1. 应用 Migration

```bash
# 本地测试
wrangler d1 execute acgti-stats --local --file=migrations/0004_stats_snapshot.sql

# 远程应用
wrangler d1 execute acgti-stats --remote --file=migrations/0004_stats_snapshot.sql
```

### 2. 部署 Cron Worker

```bash
cd cron-worker
npm install
wrangler deploy src/index.ts
```

### 3. 验证

打开 Cloudflare Dashboard，检查 Workers Logs：
- Cron Worker 应该每 15 分钟执行一次
- 日志显示 `[CRON] Updated snapshot: overview` 等信息

也可手动测试（假设 Worker URL 为 `https://acgti-stats-cron.*.workers.dev`）：
```bash
curl -X POST https://<your-worker-url>/trigger
```

## 配置调整

### 修改 Cron 频率

编辑 `cron-worker/wrangler.toml`：
```toml
[triggers]
crons = ["*/15 * * * *"]  # 改为 15 分钟
```

### 人工触发快照更新

Cron Worker 支持 `POST /trigger` 端点用于手动刷新：
```bash
curl -X POST https://<worker-url>/trigger
```

## 已有用户体验的影响

- **首次访问排行榜**：快照尚未初始化，返回空数组；首个 Cron 周期运行后正常
- **排行榜更新延迟**：从实时变更到最多 15 分钟延迟（通常远小于 15min）
- **缓存策略**：Pages Functions 仍保留 60-120 秒 HTTP 缓存，结合快照表缓存，最终用户体验变更不明显

## 注意事项

1. **Cron Worker 同 D1 绑定**：确保 `cron-worker/wrangler.toml` 中的 `database_id` 与 Pages 使用的 D1 一致
2. **快照表初始化**：首次部署后，现有 `/api/stats/*` 会返回空结果直到首个 Cron 周期完成
3. **监控日志**：建议在 Cloudflare Workers Logs 中监控 Cron 执行情况
4. **成本影响**：
   - D1 写入成本略增（每 15min 三条 INSERT/UPDATE）
   - 整体 D1 查询次数显著下降

## 后续优化空间

1. **加边缘缓存**：Pages Functions 返回时添加 `Cache-Control: public, max-age=120`（已配置）
2. **按地区缓存**：利用 Cloudflare Cache API 在全球边缘节点缓存快照
3. **细粒度快照**：若排行榜多维度，可按维度拆分快照表
4. **增量更新**：不做完整 GROUP BY，改为维护增量计数表（适合超高频写入场景）
